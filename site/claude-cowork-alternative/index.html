An open-source Claude Cowork alternative

LocalAI Cowork is for people who like the outcome-first idea of Claude Cowork but want inspectable source, local Ollama models, and control over which provider or tool receives their work.

The short version

Claude Cowork is Anthropic's polished knowledge-work agent. It can coordinate long-running work, use files and connected tools, produce professional documents, and continue remote sessions across devices. LocalAI Cowork is a smaller, early-stage Apache-2.0 Windows application. It runs the workspace on your computer and lets you choose a local Ollama model or configure compatible remote providers.

That makes LocalAI Cowork an alternative, not a drop-in clone. Choose it when source access, model choice, local execution, and modification matter more than a mature hosted ecosystem. Choose Claude Cowork when you want Anthropic's integrated models, remote continuity, broad first-party product surfaces, and commercial support.

Why local and open source can matter

A local model can be useful for drafts, classification, code, or document work that should not be sent to a hosted model. LocalAI Cowork makes that route visible: select Ollama, choose the model, then decide which files and tools the task may use. Source access also lets a team inspect storage, credential handling, release automation, and permission checks instead of relying only on product descriptions.

“Local-first” is not the same as “nothing can leave the device.” If you select OpenRouter, an OpenAI-compatible remote endpoint, a web-enabled tool, or a remote MCP server, the relevant request goes to that service. The application is designed to keep this choice explicit, and its website does not use analytics or tracking scripts.

What LocalAI Cowork does not replace

It does not provide Claude's hosted continuity, Anthropic account experience, first-party Claude models, or enterprise support. It also does not claim feature parity with Claude Cowork's document output, browser automation, mobile access, remote scheduling, or commercial administration. The current maintained release target is Windows.

Try the alternative that you can inspect

Start with a non-sensitive folder and a local Ollama model. Review the plan, tool calls, and resulting files. If you need a hosted model later, add only the provider you intend to use and understand that provider's data policy first.
